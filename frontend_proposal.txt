1. The Design Language: "Dhaka Cyberpunk" (Refined for Performance)
To keep it culturally relevant and visually striking while ensuring fast loads on mobile:

Color Palette (Optimized for Accessibility):
Background: bg-slate-950 (Deep Night—AMOLED-friendly for battery saving).
Primary Action: text-neon-green (#39FF14)—For "Profit" buttons and approvals; high contrast (AA-compliant).
Alerts/Urgency: text-rickshaw-orange (#FF5F1F)—For stockouts/risks; paired with haptic feedback on mobile.
Accents: border-jamdani-teal (#008080)—For borders and highlights; subtle gradients for depth.
New Addition: Neutral Text: text-slate-300—For body copy, with dark mode toggle (auto-detects system preference).

Typography (Refined for Multilingual Support):
Headings: Inter (Bold, clean—fast-loading Google Font).
Bangla Text: Hind Siliguri (Modern, legible—fallback to system fonts for offline).
Refinement: Auto-toggle between Bangla/English based on user input; font sizes scale dynamically (min 16px for readability).

Visual Motifs (Enhanced for Innovation):
Glassmorphism: Translucent cards (bg-slate-900/50 backdrop-blur-md) for a "cyber" overlay feel.
Glow Effects: Pulsing borders on AI-active elements (e.g., during simulations)—use CSS animations for low CPU impact.
New Refinement: Offline Fallbacks—Cached motifs (e.g., static icons) load if internet drops; progressive enhancement for PWA install prompt.
Icons: Custom set inspired by Deshi elements (e.g., rickshaw for loading, teacup for haggling)—use SVG for sharpness.

Global UX Principles (Added for Refinement):
Mobile-First: All layouts use flex/grid for responsiveness; test on 360px width (common in BD).
Accessibility: ARIA labels for voice elements; screen reader support for cards (e.g., "Trend Alert: Green Sarees"); voice navigation via Web Speech API.
Performance: Lazy-load images/videos; aim for <2s load time; use Next.js image optimization.
Ethics Integration: Every insight card includes a tappable "Why?" icon (pops up XAI explanation, e.g., "Based on TikTok trends + ethical pricing check").

3. Implementation Guide (Next.js + Tailwind) – No Code, Just Strategy

Project Structure: Root with /app (pages), /components (reusable like InsightCard, VoiceWaveform), /lib (utils for Supabase, animations).
Key Libraries: Tailwind for styling; Framer Motion for glows/animations; Chart.js for graphs; react-qr-code for onboarding; Web Speech API for voice.
Backend Ties: Use FastAPI endpoints (e.g., /risk-score for COD); Supabase for auth/realtime; Kafka events for live feeds (mock for MVP).
Testing/Refinement: Prototype flows in Figma; test on real devices (Android prevalent in BD); measure engagement (e.g., time to first action <10s).
Scalability: Modular components for easy Phase 4 expansion (e.g., add Yoruba NLP toggles).
